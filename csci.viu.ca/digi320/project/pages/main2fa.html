چطور پروتوکال اس اس اچ نصب کنیم؟

مقدمه

پوسته امن، یک پروتکل رمزگذاری شده است که برای مدیریت و ارتباط با سرورها مورد استفاده قرار می گیرد. هنگام کار با سرور دبیان شانس شما بیشتر وقت خود را در یک جلسه ترمینال متصل به سرور خود از طریق پوسته امن صرف خواهد کرد.

در این راهنما، ما بر روی تنظیم کلید های SSH برای نصب وانیلی Debian 9 تمرکز می کنیم. کلیدهای SSH یک راه آسان و ایمن برای ورود به سرور شما هستند و برای همه کاربران توصیه می شود.

پیش نیاز

شما باید به sudo دسترسی داشته باشید.

مرحله اول — ساخت کلید RSA

در مرحله اول باید در ماشین مشتری کلید را بسازید. 

    $ ssh-keygen -t rsa

به طور پیش فرض ssh-keygen یک جفت کلید RSA 2048 بیتی ایجاد می کند که برای اکثر موارد استفاده به اندازه کافی امن است (شما ممکن است به صورت اختیاری در پرچم-b 4096 برای ایجاد یک کلید بزرگتر 4096 بیتی منتقل کنید).

پس از ورود به فرمان، باید خروجی زیر را ببینید:

    Output 

    Generating public/private rsa key pair.

    Enter file in which to save the key (/your_home/.ssh/id_rsa):

برای ذخیره جفت کلیدی در پوشه .ssh / subdirectory در دایرکتوری خانگی خود وارد کنید، یا مسیر دیگری را مشخص کنید.

اگر قبلا یک جفت کلید SSH ایجاد کرده اید، ممکن است خروجی زیر را مشاهده کنید: 

    Output 

    /home/your_home/.ssh/id_rsa already exists. 

    Overwrite (y/n)?

اگر انتخاب می کنید کلید را روی دیسک بازنویسی کنید، دیگر نمی توانید با استفاده از کلید قبلی تایید هویت کنید. هنگام انتخاب بله بسیار مراقب باشید، زیرا این یک فرآیند مخرب است که قابل برگشت نیست.

ممکن است خروجی زیر را مشاهده کنید: 

    Output

    Enter passphrase (empty for no passphrase):

شما می توانید از عبارت عبور استفاده کنید. ورود به یک عبارت عبور مزایای خود را دارد: امنیت کلید، بدون توجه به اینکه چگونه رمزگذاری شده، هنوز هم بستگی به این واقعیت دارد که آن را برای هیچکس قابل مشاهده نیست. باید یک کلید خصوصی حفاظت شده با رمز عبور به اختیارات کاربران غیر مجاز وارد شود تا بتوانند از عبارت عبور استفاده کنند و برخی از زمان اضافی را به کاربر هک تبدیل کنند. البته، تنها داشتن یک عبارت عبور، باید آن را در هر بار که از جفت کلید استفاده می کنید تایپ کنید.

ممکن است خروجی زیر را مشاهده کنید: 

    Your identification has been saved in /your_home/.ssh/id_rsa.

    Your public key has been saved in /your_home/.ssh/id_rsa.pub.

    The key fingerprint is:

    SHA256:qFN5n0JubGEx6tVSDfGGxa7N+AA5hZyZDaey8EWbTnk root@instance-1 

    The key's randomart image is: 
    
    +---[RSA 2048]----+
    |       .oB+o.    |
    |       .**o*.    |
    |    . . XoE.+    |
    |     o X+* ..    |
    |      B So.=     |
    |     + B ++.o    |
    |    o . * oo     |
    |     . o .  .    |
    |                 |
    +----[SHA256]-----+

مرحله دوم — ویرایش فایل sshd_config

    sudo nano /etc/ssh/sshd_config

    PermitRootLogin prohibit-password to PermitRootLogin yes 

    PasswordAuthentication no to  PasswordAuthentication yes

مرحله سوم — ری استارت ssh 

    sudo service ssh restart

حالا شما میتوانید با هر نرم افزار پروتوکل انتقال فایل نظیر FileZilla و Cyberduck فایل ها را انتقال دهید. 

    Host:  your_server_ip_address 

    Username:  root 

    Password:  your password 

    Port:  21 (or leave it blank)

شما می توانید با استفاده از ترمینال ssh به سرور متصل شوید: 

    ssh root@your_server_ip_address

یاد بگیرید که چطور کاربر جدید بسازید.
