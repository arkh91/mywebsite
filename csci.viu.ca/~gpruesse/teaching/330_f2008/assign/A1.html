CSCI 330 Fall 2008: Assignment 1

Due Mon, Sept 29, 4:30 pm.

  1. Write a grammar for C struct statments. Use our favourite notation (->, not ::=).
  2. Add the operation "mod" to the unambiguous grammar of Example 3.4 (given in class under the heading, "An Unambiguous Grammar for Expressions"). The mod operation should be given greater precedence than +, but less than *, so A + B mod C is A + (B mod C), but A*B mod C is (A*B) mod C. Your grammar should be unambiguous. Is it left recursive?
    4. Given the grammar above, give a parse tree for A = B mod (C + A). How many leftmost derivations are there for the above assignment?
  4. Prove that the following grammar is ambiguous:
    < S > → <E >
    <S > → <F >
    <E > → <E > + <E >
    <E > → <F >
    <F > → <F > * <F >
    <F > → a
    <F > → b
    <F > → c

  5. Describe the language given by the following grammar:
    <S> → a<A>
    <A> → a<A> | <B>
    <B> → b<B> | b

    Is it ambigous? Argue your answer, briefly.

  6. Write a grammar for the strings of a's and b's of length greater than zero that read the same backwards as forwards.
  7. If value is an attribute in an attribute grammar for <arithmetic expression>, would you expect it to be synthesized or inherited?
  8. For stack-dynamic strings, if max_possible_length is an attribute, would you expect it to be synthesized or inherited? If current_length is an attribute, would you expect it to be synthesized or inherited, and when (i.e., in which side of an assignment operation)?
  9. P 241 (Ed 8) or 244 (Ed 7), question 8.
  10. P 243 (Ed 8) or 246 (Ed 7), question 11.
