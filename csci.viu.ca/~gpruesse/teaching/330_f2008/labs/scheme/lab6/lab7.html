  1. Add to your file of tree functions the function build-tree, which takes a list and builds a tree of the elements of the list. The tree should be in Binary Search Tree order -- keys less than the root are in the left subtree, and keys greater than the root are in the right subtree.

    You may want to write, as a helper function, a function that inserts a new key into a given tree. If you have insert and fold, how can you write build-tree? Fold is given as 

    ; fold operator f over l, where f has identity id
(define fold (lambda (f l id)
  (if (null? l) id
      (f (car l) (fold f (cdr l) id)))))
    Fold takes a function f, a list, and the identity value for the function -- for example, the identity value of + is 0 (because `+0' is the identity function) and the identity value of * is 1 (because `*1' is the identity function) -- and returns the result of applying f iteratively to the list of elements. For example, (fold + `(5 2 1) 0) is (+ 5 (+ 2 (+ 1 0))).

  2. Add to your file of tree functions the function inorder, which takes a tree and makes a list of the data in the tree in inorder traversal order. (This may be similar to your traverse function.)
    3. For example, if you were to call (inorder (build-tree `(3 9 77 2 19 1 6))) mzscheme would respond with (1 2 3 6 9 19 77)
